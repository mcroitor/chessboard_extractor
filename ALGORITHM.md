# Описание алгоритма

Алгоритм нахождения шахматных диаграмм основан на факте, что шахматная диаграмма это довольно крупный квадрат. На качественных сканах можно отслеживать шахматный паттерн (64 квадрата, чередующихся по цвету), но на сканах низкого качества это может быть проблемой. Поэтому алгоритм ориентирован на нахождение крупных "почти" квадратов, которые могут быть шахматными диаграммами.

1. На вход алгоритму подается скан страницы в виде изображения.
2. Алгоритм выполняет предварительную обработку изображения, которая может включать в себя:
   - Преобразование изображения в градации серого.
   - Применение фильтров для уменьшения шума.
3. Затем алгоритм выполняет поиск контуров на изображении, используя методы компьютерного зрения (например, алгоритм Канни для обнаружения границ).
4. Для каждого контура строится bounding rectangle (ограничивающий прямоугольник).
5. Выкидываем bounding rectangle, которые слишком малы или слишком велики, так как они вряд ли будут шахматными диаграммами.
6. Оставшиеся прямоугольники объединяются, если они пересекаются (этап оптимизации), чтобы собрать цельную область диаграммы из нескольких контуров.
7. Для оставшихся кандидатов выполняется проверка на "почти квадрат" (ширина примерно равна высоте).
8. Для оставшихся кандидатов алгоритм может дополнительно проверять наличие шахматного паттерна внутри них, если качество изображения позволяет это сделать.
9. Сохраняются только вырезанные ROI-области, которые с большой вероятностью являются шахматными диаграммами.
