# chessboard_extractor

`chessboard_extractor` — небольшой инструмент для извлечения шахматных диаграмм со сканов страниц.

Алгоритм ищет крупные почти-квадратные области (кандидаты на диаграммы). При неправильных настройках возможны ложные срабатывания или пропуски.

## Использование

Пример запуска:

```cmd
extractboards --input=<input_dir> --output=<output_dir> --format=<page_extension>
```

Также можно настроить параметры через `config.ini`.

```ini
# размер ядра blur
blur_size        = 21

blur_standard   = 0
blur_median     = 0
blur_gaussian   = 1

# замыкание небольших разрывов на границах после Canny
close_gaps      = 1
morph_kernel_size = 5
morph_iterations  = 1

contour_delta    = 50

# формат входных страниц
format   = png

# допуск в пикселях для сравнения размеров/квадратности
gap      = 50

# объединять близкие прямоугольники во время optimize
merge_gap = 8

# входная папка
input    = .\in\

# максимальный размер диаграммы
max_board_size   = 700

# минимальный размер диаграммы
min_board_size   = 500

# выходная папка
output   = .\out\

# отбрасывать не-квадратные кандидаты
remove_non_squares       = 1

# толщина рамки для markboards
thickness        = 5

# порог Canny
treshold         = 100
```

## Параметры и ожидаемый эффект

| Параметр | Увеличение / включение | Уменьшение / выключение |
|---|---|---|
| `blur_size` | Больше сглаживания, меньше шумов, но тонкие границы могут исчезать | Больше деталей, но больше шумовых контуров |
| `blur_gaussian` | Мягкое подавление шума, обычно хороший базовый вариант | Больше «сырых» границ и шума |
| `blur_median` | Лучше подавляет дефекты печати и одиночные шумовые точки | Ниже устойчивость к точечным артефактам |
| `blur_standard` | Сильнее усредняет, может «съедать» слабые границы | Лучше сохраняет локальные детали |
| `treshold` | Canny строже: меньше границ и контуров | Canny чувствительнее: больше границ и ложных контуров |
| `contour_delta` | Сильнее упрощает контур, грубее геометрия | Точнее форма, но больше шумовых деталей |
| `min_board_size` | Отсекает мелкие кандидаты, меньше ложных срабатываний | Пропускает больше мелких кандидатов, больше шума |
| `max_board_size` | Разрешает более крупные кандидаты | Отсекает крупные области |
| `remove_non_squares` | Оставляет только почти квадратные области | Оставляет все формы (полезно для диагностики) |
| `gap` | Больше допуск в проверке «почти квадрат» | Более строгая проверка квадратности |
| `close_gaps` | Включает morphological closing после Canny, помогает при разрывах рамки | Разрывы остаются, часть диаграмм может не находиться |
| `morph_kernel_size` | Склеивает более крупные разрывы, но может переобъединять | Более мягкая склейка, меньше риск слияний |
| `morph_iterations` | Усиливает эффект склейки разрывов | Ослабляет эффект склейки |
| `merge_gap` | Объединяет близкие прямоугольники даже без прямого пересечения | Объединяет только пересекающиеся/очень близкие |
| `thickness` (`markboards`) | Толще красная рамка на отладочном выводе | Тоньше рамка |

## Практические пресеты

Базовый пресет для сканов плохого качества с разрывами рамки:

```ini
blur_size = 21
blur_gaussian = 1
blur_median = 0
blur_standard = 0

close_gaps = 1
morph_kernel_size = 5
morph_iterations = 1
merge_gap = 8
```

Если разрывы рамки сильные:

- увеличьте `morph_kernel_size` до `7`
- при необходимости увеличьте `morph_iterations` до `2`
- увеличьте `merge_gap` до `10..12`

Если появляются лишние объединения:

- сначала уменьшите `merge_gap`
- затем уменьшите `morph_kernel_size` или `morph_iterations`

## Диагностика по симптомам

### Слишком много пропусков диаграмм

- уменьшите `treshold` (например, `100 -> 80`)
- усилите склейку разрывов: `morph_kernel_size 5 -> 7`
- увеличьте `merge_gap` (`8 -> 10`)
- временно поставьте `remove_non_squares = 0` для диагностики кандидатов

### Слишком много ложных срабатываний

- увеличьте `treshold` (`100 -> 120`)
- увеличьте `min_board_size`
- уменьшите `max_board_size`
- оставьте `remove_non_squares = 1`
- уменьшите `merge_gap` (`10 -> 6`)

### Не находится диаграмма с рваной/разорванной рамкой

- включите `close_gaps = 1`
- увеличьте `morph_kernel_size` (`5 -> 7`)
- при необходимости увеличьте `morph_iterations` (`1 -> 2`)
- если рамка распадается на части, увеличьте `merge_gap`

### Соседние объекты сливаются в один большой прямоугольник

- уменьшите `merge_gap`
- уменьшите `morph_kernel_size`
- уменьшите `morph_iterations`

### Найденный прямоугольник недостаточно квадратный

- уменьшите `gap`, чтобы сделать проверку квадратности строже
- оставьте `remove_non_squares = 1`

### Удобный сценарий отладки

- сначала запускайте `markboards`, чтобы увидеть красные прямоугольники
- меняйте по одному параметру за раз
- после каждого изменения сравнивайте результаты постранично

## Сборка

Проект использует OpenCV для поиска контуров.

Если используете MSYS2/Mingw64, установите OpenCV (например, через `pacman -Ss opencv`).

```bash
git clone git@github.com:mcroitor/chessboard_extractor.git
cd chessboard_extractor
make
```

## Рекомендации

Инструмент не универсален: при плохом качестве печати и разрывах на полях диаграммы качество детекции падает.

Перед обработкой полезно предварительно улучшить скан (выравнивание фона, повышение контраста, усиление линий).

Также рекомендуется заранее оценить реальный размер диаграмм и выставить `min_board_size`/`max_board_size` максимально узко под ваш набор страниц.
